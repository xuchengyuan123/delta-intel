----------------------------------------------------------------

  在任意目录执行：
    npx wrangler deploy --config "C:/Users/徐承远/Desktop/三角洲情报台-上传包/cloudflare/wrangler.toml"

方式 B（Cloudflare 控制台粘贴）：
  Workers & Pages → delta-intel-auth → 编辑代码 → 全选粘贴 cloudflare/worker.js → 保存部署。
  注意：控制台部署不会读 wrangler.toml，需手动在 Worker「设置 → 变量 → KV 命名空间绑定」
        里把全部 KV 绑好（USERS/SESSIONS/CODES/POSTS/ANNOUNCE/UGC/COMMENTS/ACCESS/
        PRESENCE/FRIENDS/MESSAGES/AVATARS，共 12 个，已在 v21 建好）。

★ 本版【无需新建 KV 命名空间】。

二、（可选）开启 AI 自动审核 —— 配置 3 个 Secret
----------------------------------------------------------------
若要让「发帖自动审核」和「🤖 AI 审核」按钮真正调 AI，需配置：
  wrangler secret put AI_API_KEY   # 任意 OpenAI 兼容接口的 Key（OpenAI / DeepSeek 等）
  wrangler secret put AI_API_URL   # 完整 chat completions 地址，例如：
                                    #   https://api.openai.com/v1/chat/completions
                                    #   https://api.deepseek.com/v1/chat/completions
  wrangler secret put AI_MODEL     # 模型名，默认 gpt-4o-mini（DeepSeek 填 deepseek-chat）
不配置也完全正常：发帖 / AI 审核按钮都会维持「转人工待审」，行为不变。
（注：OpenAI / DeepSeek 等大多需绑卡，你目前可能暂无；先不配，等有了再 put 即可生效。）

四、AI 审核接口用法（POST /api/admin/ugc/aimod）
----------------------------------------------------------------
需以管理员身份登录后拿到的会话令牌（Bearer），放在请求头 Authorization。

1) 批量审查全部「待审(pending)」投稿：
   curl -X POST "https://api.delta.shopping/api/admin/ugc/aimod" \
     -H "Authorization: Bearer <管理员token>" \
     -H "Content-Type: application/json" \
     -d '{"all":true}'
   返回：{ "ok":true, "processed":N, "results":[ {id,decision,status,reason}, ... ] }

2) 单条审查某个投稿（id 换成真实投稿 id，如 u:xxxx）：
   curl -X POST "https://api.delta.shopping/api/admin/ugc/aimod" \
     -H "Authorization: Bearer <管理员token>" \
     -H "Content-Type: application/json" \
     -d '{"id":"u:xxxx"}'

说明：decision=approve → 直接公开；reject → 直接拒绝；review → 转人工队列。
      命中敏感词或 AI 未配置/调用异常，一律转 review（人工）。

五、部署后自检
----------------------------------------------------------------
1) 后端 v23 是否上线（未登录应 401 而非 404）：
   curl -X PUT "https://api.delta.shopping/api/user/avatar" -H "Content-Type: application/json" -d '{"image":"x"}'
   → 返回 {"error":"未登录或会话已过期"} 即说明 Worker 已是最新版。

2) AI 审核接口是否存在（管理态未登录 401，说明路由已注册）：
   curl -X POST "https://api.delta.shopping/api/admin/ugc/aimod" -H "Content-Type: application/json" -d '{"all":true}'
   → 401 表示路由已存在（配好 AI 后发关注 behavior 变化）。

3) 前端：用总管理员登录后台，「总览与账号」分组下应出现「👥 用户管理」面板；
   「📝 用户投稿审核」面板顶部出现「🤖 用 AI 批量审核全部待审」按钮。
