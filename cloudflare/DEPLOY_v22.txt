一、部署后端 Worker（与 v21 相同，KV 已在 v21 建好，无需再建）
----------------------------------------------------------------
方式 A（命令行，推荐，用 --config 指定绝对路径，不必管当前目录）：
  在任意目录执行：
    npx wrangler deploy --config "C:/Users/徐承远/Desktop/三角洲情报台-上传包/cloudflare/wrangler.toml"

方式 B（Cloudflare 控制台粘贴）：
  Workers & Pages → delta-intel-auth → 编辑代码 → 全选粘贴 cloudflare/worker.js → 保存部署。
  注意：控制台部署不会读 wrangler.toml，需手动在 Worker「设置 → 变量 → KV 命名空间绑定」
        里把 USERS/SESSIONS/CODES/POSTS/ANNOUNCE/UGC/COMMENTS/ACCESS/PRESENCE/FRIENDS/MESSAGES/AVATARS
        全部绑好（这 11 个在 v21 已建）。

★ 本次【无需新建 KV 命名空间】——用户管理复用现有 USERS KV，好友/头像等沿用 v21 的 5 个。

----------------------------------------------------------------
二、（可选）开启 AI 发帖审核 —— 配置 3 个 Secret
----------------------------------------------------------------
不配置也完全正常：论坛发帖会维持原来的「人工待审」，行为不变。
在 cloudflare/ 目录下执行：
  wrangler secret put AI_API_KEY     # 任意 OpenAI 兼容接口的 Key（OpenAI / DeepSeek 等）
  wrangler secret put AI_API_URL     # 完整 chat completions 地址，例如：
                                      #   https://api.openai.com/v1/chat/completions
                                      #   https://api.deepseek.com/v1/chat/completions
  wrangler secret put AI_MODEL       # 模型名，默认 gpt-4o-mini（DeepSeek 填 deepseek-chat）
配置后，新发帖会由 AI 自动审核：正常→直接公开；违规→直接拒绝；敏感词/异常→转人工队列。
（注意：OpenAI / DeepSeek 等大多需绑卡，你目前可能暂无；先不配，等有了再 put 即可生效。）

----------------------------------------------------------------
四、部署后自检
----------------------------------------------------------------
1) 后端 v22 是否上线（未登录应 401 而非 404）：
   curl -X PUT "https://api.delta.shopping/api/user/avatar" -H "Content-Type: application/json" -d '{"image":"x"}'
   → 返回 {"error":"未登录或会话已过期"} 即说明 Worker 已是最新版。

2) 用户管理端点（仅总管理员 / 用户管理分管理员可用，未登录 401）：
   curl "https://api.delta.shopping/api/admin/members"
   → 401 表示路由已存在（配置好 AI 审核后发帖行为变化，可在后台「用户投稿审核」看 AI 自动通过/拒绝记录）。

----------------------------------------------------------------
五、使用要点
----------------------------------------------------------------
· 创建「用户管理」分管理员：后台「分管理员管理」→ 角色卡片选「👥 用户管理管理员」
  （或手动在面板清单里只勾「用户管理」）→ 填邮箱密码创建。该分管理员登录后
  左侧只会出现「用户管理」面板，看不到其它模块。
· 其它分管理员（赛季任务/音乐等）依旧看不到「用户管理」面板，符合你的要求。
· 分管理员权限现在按「所选面板」精确控制：在创建页勾哪些，他就只能进哪些。
