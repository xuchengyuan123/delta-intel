一、本次新增（相对 v35）
----------------------------------------------------------------
1. 【后台面板】「安全防护 · 境外审核」
   - 位置：后台左侧「总览与账号」分组下，图标 🛡。总管理员与「安全运维(secops)」分管理员可见。
   - 功能：
     · 显示「10 层应用防火墙已启用」状态说明。
     · 🌍 境外请求待审核列表：所有非中国大陆（country != CN）的访问 IP 自动进入此队列，
       展示 IP / 国家 / 首次&最近访问时间 / UA / 示例请求路径。
     · 每个待审核项可一键【通过】（放行，以后不再拦截该 IP 的审核标记）或【封禁】。
     · 🚫 已封禁 IP 列表，支持【解封】。
     · ➕ 手动封禁：输入任意 IP 直接封禁（封禁后该 IP 访问 Worker 一律 403）。
   - 数据每 60 秒自动刷新（页面切到后台时暂停，回到前台恢复）。

2. 【Worker 10 层应用防火墙】（在 cloudflare/worker.js 的 securityGate 中实现）
   第 1 层  IP 黑名单（管理员封禁的 IP 直接 403）
   第 2 层  HTTP 方法白名单（仅放行 GET/POST/PUT/DELETE/OPTIONS/HEAD/PATCH）
   第 3 层  User-Agent 基础校验（空 / 过短 UA 拦截）
   第 4 层  已知恶意 Bot / 扫描器 / 攻击工具 UA（curl/wget/sqlmap/nmap/nikto/wpscan… 拦截）
   第 5 层  请求体大小限制（POST/PUT > 64KB 拒绝，防大包耗尽资源）
   第 6 层  路径穿越 / SQL 注入 / XSS 特征过滤（path + query + UA 命中即拦截）
   第 7 层  单 IP 速率限制（内存滑动窗口：中国大陆 120 次/分钟，境外 40 次/分钟，超则 429）
   第 8 层  登录防暴力破解（后台登录连续错误 5 次临时锁定 15 分钟）
   第 9 层  国别策略（境外请求写入后台审核队列，交由管理员放行/封禁；不自动封禁）
   第 10 层 敏感/高频端点节流（排行榜、访问日志、战绩等昂贵 GET 接口额外收紧）

   说明：Cloudflare 本身已在网络层（L3/L4）免费提供 DDoS 缓解；
        本 10 层是「应用层(L7)」防护，挡住爬虫、扫描器、注入、爆破、滥用轮询等。
        若遭遇大规模 volumetric DDoS，可在 Cloudflare 控制台开启「Under Attack Mode」。

3. KV 额度友好设计（避免重蹈 v35 前 KV 超限）
   - 黑名单读取走内存缓存（单 IP 60 秒内只查 1 次 KV），其余层纯内存，不写 KV。
   - 境外审核队列：同一 IP 至多写 1 次 KV（内存去重），不随请求数增长。
   - 登录锁定仅在「达阈值」时写 1 次 KV。
   - 免费 KV 额度（1000 写 / 10 万读 每天）日常足够。

二、后端部署（Worker）
----------------------------------------------------------------
1. 打开 Cloudflare 控制台 → Workers & Pages → 你的 Worker（delta-intel-auth）。
2. 用本文件夹最新的 cloudflare/worker.js 全文覆盖并部署（已含 v36 防火墙与审核接口）。
3. 变量 / Secret 确认：
   · 已有：GH_TOKEN、SUPER_EMAIL、SUPER_PASSWORD、REPO、DEV_MODE
   · 建议：DF_STATS_KEY（战绩接口，可选）
   · 可选新增 KV 绑定 SECURITY：
     - 在 Worker 的「设置 → 变量 → KV 命名空间绑定」新增绑定，变量名 SECURITY，
       指向一个新建的 KV 命名空间（例如 delta-intel-security）。
     - 不配置也能用：未绑定 SECURITY 时，黑名单/审核数据会自动复用 ACCESS 命名空间。
     - 建议单独建，便于清理与管理安全数据，不影响访问日志。
4. 部署后建议先在浏览器访问一次站点，确认无 500（若有报错，看 Worker 日志）。

四、如何使用「境外请求审核」
----------------------------------------------------------------
1. 以总管理员（或安全运维分管理员）身份登录后台（admin.html）。
2. 左侧「总览与账号 → 🛡 安全防护 · 境外审核」。
3. 看到待审核的境外 IP 后：
   - 若是正常访客/朋友 → 点【通过】放行。
   - 若是可疑/攻击来源 → 点【封禁】，该 IP 立即被第 1 层墙拦截（403）。
4. 也可在「手动封禁 IP」框输入已知恶意 IP 直接封禁。

五、重要提醒
----------------------------------------------------------------
· 防火墙与审核逻辑全部在 Worker 端，需先完成「二、后端部署」才会生效。
· 仅中国大陆（CN）放行；若你本人或朋友在境外访问，会进入待审核队列，
  在后台【通过】一次即可（之后该 IP 不再被标记）。
· 登录防爆破仅针对后台登录接口（/api/admin-login），连续错 5 次锁 15 分钟。
· 免费 KV 仍可能在大流量下触顶；如访问日志不需要，可注释 worker.js 中
  recordAccess 调用（参考 v35 部署说明的「KV 额度优化」章节）。
