# 三角洲情报台 · 分管理员后端部署指南

> 目的：让「分管理员」能用 **邮箱/手机号 + 密码（或验证码）** 登录后台，各自只看到自己负责的模块。
> 纯 GitHub Pages 是静态站点，没有服务器，所以必须再挂一个 **Cloudflare Worker** 当后端。
> 本项目已写好前端调用逻辑（`admin.html` 的 `wkFetch`），缺的只是这个后端 —— 本指南把它补齐并说明如何上线。

---

## 一、整体原理（先看懂，少踩坑）

```
浏览器(admin.html)  ──https──>  Cloudflare Worker  ──https──>  GitHub API
  分管理员登录/读写数据          账号校验 + 会话管理             代你提交 data.json
                               KV 存账号/会话/验证码          用 GH_TOKEN 写仓库
```

- **分管理员不需要自己的 GitHub Token**：Worker 持有 `GH_TOKEN`，替所有人把改动提交到仓库。
- **总管理员有两种登录方式**：
  - 「总管理员·Token」标签：直接用 GitHub Token 写仓库（**不需要 Worker**，但也不能管分管理员）。
  - 「分管理员·账号」标签：用 **邮箱+密码** 登录 Worker（**部署本后端后才有**）。要创建/管理分管理员，必须用这种方式以总管理员身份登录。
- **验证码**：默认 `DEV_MODE=true`，验证码会直接显示在后台界面上（零成本、适合个人站）。想要真发邮件，再接 EmailJS（见第七节）。

---

## 二、准备工作（你自己的账号）

1. **Cloudflare 账号**：免费，去 https://dash.cloudflare.com/sign-up 注册。
2. **GitHub Token**（给 Worker 写仓库用）：
   - GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token。
   - 勾选 `public_repo`（公开仓库）或 `repo`（私库）；有效期设长一点（如 90 天或 No expiration）。
   - 复制下来，**形如 `ghp_xxxx...`**，妥善保管。
3. 想好一个**总管理员邮箱 + 密码**（≥6 位），例如 `admin@你的域名.com` / `你的密码`。
   - 这个账号第一次访问 Worker 时会**自动创建**，就是总管理员（role=super）。

## 三、在本地准备好文件

后端代码已放在上传包的 `cloudflare/` 目录，里面有：
- `worker.js` —— Worker 主程序（已实现全部接口）。
- `wrangler.toml` —— 配置文件（KV 绑定 + 变量）。
- `.dev.vars.example` —— 本地调试用的变量示例。

把整个 `cloudflare/` 文件夹复制到你的电脑上某个目录（比如 `D:\delta-worker\`）。

> **关于 `.dev.vars.example`**：该文件列出了本地/调试需要的全部变量名，方便你复制为 `.dev.vars` 后在本地 `wrangler dev` 联调（见下方「可选 · 本地预览」）。`.dev.vars` 含密钥，**切勿提交到仓库**。直接部署线上则用第六节的 `wrangler secret put`，不需要这个文件。

### （可选）本地预览测试

不想马上部署到公网、想先在本地验证后端逻辑，可以（仍需先完成第五节创建 3 个 KV 命名空间并把真实 id 填进 `wrangler.toml`）：

```bash
cp .dev.vars.example .dev.vars      # 填入真实值
wrangler dev                        # 本地起 http://127.0.0.1:8787
```

本地起好后，把后台「分管理员·账号」标签里的 Worker 地址填成 `http://127.0.0.1:8787` 即可联调。注意 `wrangler dev` 的 KV 是本地临时存储，与线上 KV 不互通；验证完仍需用 `wrangler deploy` 部署到公网才能对外使用。

## 四、安装并登录 wrangler（Cloudflare 命令行）

需要 Node.js（已装的话跳过）：https://nodejs.org 下载安装。

打开命令行（CMD / PowerShell / 终端），进入 `cloudflare/` 目录：

```bash
cd cloudflare
npm install -g wrangler
wrangler login          # 浏览器弹出，登录你的 Cloudflare 账号并授权
wrangler whoami        # 能看到你的账号名就说明登录成功
```

---

## 五、创建 3 个 KV 命名空间

Worker 用 KV 存「账号 / 会话 / 验证码」。依次执行下面三条命令，**每条都会返回一个 id**，复制下来：

```bash
wrangler kv namespace create USERS
wrangler kv namespace create SESSIONS
wrangler kv namespace create CODES
```

例如返回：
```
🆕  Created namespace USERS with id "a1b2c3d4e5f60718293a4b5c6d7e8f90"
```
把那串十六进制分别填进 `wrangler.toml` 里的占位符 `__USERS_ID__`、`__SESSIONS_ID__`、`__CODES_ID__`。

---

## 六、填入密钥（敏感信息，不要写进文件）

在 `cloudflare/` 目录下，逐条执行（每条会让你输入值，粘贴后回车）：

```bash
wrangler secret put GH_TOKEN
wrangler secret put SUPER_EMAIL
wrangler secret put SUPER_PASSWORD
```

- `GH_TOKEN` = 第二步拿到的 GitHub Token。
- `SUPER_EMAIL` = 你的总管理员邮箱。
- `SUPER_PASSWORD` = 你的总管理员密码。

> 这些密钥存在 Cloudflare 侧，**不会出现在代码或仓库里**，比写在 wrangler.toml 安全。

`wrangler.toml` 里的 `REPO` 和 `DEV_MODE` 已经填好（仓库名 `xuchengyuan123/delta-intel`，开发模式开），一般不用改。

---

## 七、（可选）接 EmailJS 真发验证码

不想在界面上看验证码、想真发邮件，就去 https://www.emailjs.com 免费注册，建一个发码模板，然后：

```bash
wrangler secret put EMAILJS_SERVICE_ID
wrangler secret put EMAILJS_TEMPLATE_ID
wrangler secret put EMAILJS_PUBLIC_KEY
```

并把 `wrangler.toml` 里的 `DEV_MODE = "true"` 改成 `"false"`。
**不接也能用**：`DEV_MODE=true` 时验证码直接回显到后台「发送中…」提示里。

---

## 八、部署上线

在 `cloudflare/` 目录执行：

```bash
wrangler deploy
```

成功后命令行会给出一个地址，类似：
```
https://delta-intel-auth.<你的子域>.workers.dev
```
**这就是「Worker 地址」**，复制它。

## 九、在后台里启用

1. 打开你的站点后台 `https://xuchengyuan123.github.io/delta-intel/admin.html`。
2. 登录页切到 **「分管理员·账号」** 标签。
3. 在「Cloudflare Worker 地址」输入框粘贴上一步的 `*.workers.dev` 地址（输完会自动保存）。
4. 用**总管理员邮箱 + 密码**登录（就是第六步设的 `SUPER_EMAIL` / `SUPER_PASSWORD`）。
5. 进入后左侧选 **「分管理员管理」**：
   - 先看 📖 使用说明；
   - 点选一个角色卡片（如「音乐管理员」），填邮箱、密码（≥6 位）、可选手机号，点「创建分管理员」；
   - 下方「现有账号」会列出已创建的账号。
6. 把账号交给对应的人，他们用各自邮箱/手机号 + 密码（或点「获取验证码」）登录，只能看到自己负责的模块。

## 十、常见问题

- **登录提示「请先填写 Worker 地址」**：后台「分管理员·账号」标签里那个地址框是空的，填好再登录。
- **「该账号不存在」**：分管理员必须先由总管理员在「分管理员管理」里创建，才能去登录页获取验证码 / 登录。
- **Worker 地址谁都能访问，安全吗？** 接口都要求登录会话或总管理员权限；`GH_TOKEN` 只存在 Cloudflare 侧，不会暴露给浏览器。但 `*.workers.dev` 是公网地址，建议上线后在 Cloudflare 面板给 Worker 加一个自定义域名 + 访问控制（可选）。
- **不想用分管理员了？** 直接用「总管理员·Token」标签 + GitHub Token 登录即可，完全不依赖 Worker。
- **改了代码要重新上线？** 在 `cloudflare/` 目录再跑一次 `wrangler deploy`。

## 十二、大陆网络必读：免费 `*.workers.dev` 被墙，请绑自定义域名

> **实测（2026-07-12）**：`curl https://delta-intel-auth.<子域>.workers.dev` 与浏览器均报 `ERR_CONNECTION_TIMED_OUT`。DNS 探测显示该域名被解析到 **Facebook/Meta 的假 IP**（`157.240.8.50` / `2a03:2880:...`），连接必然超时。这是 **GFW 对 `*.workers.dev` 的 DNS 污染**，不是配置错误——免费 `*.workers.dev` 在大陆基本连不上。

### 现象
- 后台「分管理员·账号」点「发送验证码」报 `net::ERR_CONNECTION_TIMED_OUT`。
- `curl https://...workers.dev` 也超时；`nslookup` 解析到的不是 Cloudflare IP（`104.16/172.67/188.114`），而是 `157.240.x.x` 等 Meta 段。

### 解决：给 Worker 绑一个你自己域名的子域
走 Cloudflare 真 CDN，解析到 Cloudflare 自己的 IP，国内可通。

1. **注册一个便宜域名**：`.top` 约 ¥9–20/年（阿里云/腾讯云均可，需实名）；`.cn` 也行但要实名认证。
2. **接入 Cloudflare**：dash.cloudflare.com → Add a Site → 填域名 → Free 套餐 → 按提示把注册商处的 NS 改成 Cloudflare 给的两个（如 `xxx.ns.cloudflare.com`）。
3. **等 NS 生效**：几分钟到几小时。`nslookup 你的域名` 解析到 Cloudflare IP 即成功。
4. **改 `wrangler.toml`**，末尾加：
   ```toml
   [custom_domain]
   zone_name = "你的域名"          # 例如 delta-intel.top
   custom_domain = "api.你的域名"  # 例如 api.delta-intel.top
   ```
5. **重新部署**：在 `cloudflare/` 目录 `wrangler deploy`。
6. **后台填新地址**：「分管理员·账号」标签里把 Worker 地址改为 `https://api.你的域名`，保存。

> 注意：custom_domain 要求该域名已是 Cloudflare 管理的 zone（NS 已切换且生效），否则 `wrangler deploy` 会报 zone 找不到。
