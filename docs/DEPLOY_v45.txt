# 三角洲情报台 V45 部署说明（零代码开发工作台）

本版是重大功能版：后台「开发工作台」升级为零代码建站器（应用→模块→页面 + 全模板）。
分两部分部署，**都由你（用户）在浏览器/本机执行，AI 只改了本地文件，没有动 GitHub**。

=====================================================
一、先部署 Worker（必须，否则新接口不生效）
=====================================================
V45 在 worker.js 里新增/修改了这些接口：
  - GET/PUT /api/admin/apps      （工作台应用数据，存 WORKBENCH KV）
  - POST /api/app/submit         （表单收集页提交，公开）
  - GET  /api/admin/app-submissions （后台查看表单提交，super）
把旧接口 /api/admin/works 升级为 /api/admin/apps（KV key 从 "works" 改为 "apps"）。

步骤（二选一）：
A. 控制台粘贴（推荐，快）：
  1. 打开 Cloudflare 控制台 → Workers & Pages → 你的 Worker `delta-intel-auth`（自定义域 api.delta.shopping）。
  2. 编辑代码，把 `cloudflare/worker.js` 的【全部内容】粘贴覆盖，保存并部署。
B. wrangler 部署：
  在 `cloudflare/` 目录执行 `wrangler deploy`（需已登录且 wrangler.toml 配置正确）。

【关键】绑定 WORKBENCH KV（工作台数据 + 表单提交都依赖它）：
  1. Cloudflare 控制台 → Workers & Pages → 你的 Worker → Settings → Variables → KV Namespace Bindings → Add binding。
  2. Variable name 填 `WORKBENCH`，绑到一个 KV 命名空间（没有就先 Create 一个）。
  3. 保存。不绑也能跑，但工作台应用数据/表单提交会无法持久化（接口会返回明确提示）。

=====================================================
二、再上传整包到 Pages（GitHub Pages）
=====================================================
把本地 `三角洲情报台-上传包/docs/` 目录的【全部内容】上传/覆盖到你的 Pages 仓库根目录
（即 xuchengyuan123/delta-intel 仓库根，站点文件约定在仓库的 docs/ 子目录）。
上传方式按你习惯（Git 推送或 GitHub 网页拖拽覆盖），AI 不参与。

=====================================================
三、硬刷新
=====================================================
打开网站首页 → Ctrl+F5（或清缓存刷新），让新版 Service Worker（delta-intel-v45）接管并重新预缓存。

=====================================================
四、工作台怎么用（零代码建站）
=====================================================
1. 后台左侧「开发工作台」→ 应用管理 → ＋新建应用（填名称/图标/类型/状态，可勾「进前台导航」）。
2. 进入应用 → ＋新建模块（如「报名通道」）。
3. 进入模块 → ＋新建页面，选模板并填内容：
   - 内容展示：空白 / 列表 / 图文 / 链接集合 / 卡片网格 / 图库
   - 表单收集：自助增删字段（文本/多行/数字/邮箱/电话/单选/多选），访客提交存后台，可在「提交」里查看/导出 CSV
   - 数据看板：勾选要展示的指标（站点名/版本/更新数/浏览数/论坛帖数/在线数等），实时读 data.json
4. 点「保存并生成」→ 页面 HTML 经 Worker 写入站点根，前台可访问。
5. 应用详情页可「🌐 生成应用首页」（列出该应用下所有模块与页面，作为访客入口）。
6. 勾了「进前台导航」的应用/页面，会出现在主站左侧「应用」分组，点击直达。
7. 已生成的页面可再次「编辑」→ 改内容 →「保存并生成」覆盖更新。

注意：V44 及以前工作台数据存在 WORKBENCH KV 的 "works" key；V45 改为 "apps" key，
旧数据不自动迁移，重新建应用即可（不影响其它功能）。
