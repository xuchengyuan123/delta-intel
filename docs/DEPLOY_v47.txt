# 三角洲情报台 V47 部署说明（可视化零代码拼搭器 + 安全加固）

V47 = V45（零代码工作台） + V46（安全加固，未升版本号） + V47（工作台升级为可视化拼搭器）。
分两部分部署，**都由你（用户）在浏览器/本机执行，AI 只改了本地文件，没有动 GitHub**。

=====================================================
一、先部署 Worker（必须，否则新接口/安全修复不生效）
=====================================================
V46/V47 在 worker.js 里：
  - 修复路径穿越高危漏洞（sanitizeRepoPath 收敛到 docs/）
  - /api/app/submit 增加 ID 白名单、字段上限、单 IP 限流
  - /api/admin/data 写入前 JSON 校验
  - 验证码改用 Web Crypto 强随机
  - 战队 /api/teams* 全套接口（创建/列表/我的/详情/加入/退出/战绩/置顶/解散）

步骤（二选一）：
A. 控制台粘贴（推荐，快）：
  1. 打开 Cloudflare 控制台 → Workers & Pages → 你的 Worker `delta-intel-auth`（自定义域 api.delta.shopping）。
  2. 编辑代码，把 `cloudflare/worker.js` 的【全部内容】粘贴覆盖，保存并部署。
B. wrangler 部署：
  在 `cloudflare/` 目录执行 `wrangler deploy`（需已登录且 wrangler.toml 配置正确）。

【关键】绑定 KV（变量名必须一致，创建 KV 命名空间【不需要实名】，免费邮箱账号即可）：
  Cloudflare 控制台 → Workers & Pages → Worker → Settings → Variables → KV Namespace Bindings → Add：
  - USERS / SESSIONS / CODES / POSTS / ANNOUNCE / UGC / COMMENTS / ACCESS
  - PRESENCE / FRIENDS / MESSAGES / AVATARS / WORKBENCH / TEAMS / LB / SECURITY
  （未用的可暂不绑，对应功能会返回明确提示，不会崩溃）

【战队能用起来的关键】务必绑定 TEAMS：
  Variable name 填 `TEAMS`，绑到一个 KV 命名空间（没有就先 Create 一个）。
  未绑时打开战队页会显示「加载失败：KV 命名空间 TEAMS 未绑定到 Worker…」——这是正常提示，不是 bug，绑上即可用。

=====================================================

注意本次版本号已升到 v47：所有 `?v=45` → `?v=47`，sw.js `CACHE="delta-intel-v47"`，data.json changelog 已加 v46/v47 条目。
上传方式按你习惯（Git 推送或 GitHub 网页拖拽覆盖），AI 不参与。

=====================================================
三、硬刷新
=====================================================
打开网站首页 → Ctrl+F5（或清缓存刷新），让新版 Service Worker（delta-intel-v47）接管并重新预缓存。

=====================================================
四、工作台怎么用（可视化零代码拼搭器）
=====================================================
1. 后台左侧「开发工作台」→ 应用管理 → ＋新建应用（填名称/图标/类型/状态，可勾「进前台导航」）。
2. 进入应用 → ＋新建模块（如「报名通道」）。
3. 进入模块 → ＋新建页面：
   - 选「🧩 可视化拼搭」：从组件库点选添加 12 种组件（标题/段落/图片/按钮/分割线/间距/列表/卡片网格/表格/表单/数据看板/数据视图）→ 点「配置」改属性 → 拖拽排序 → 保存即生成页面。
   - 或选「📐 经典模板」：空白/列表/图文/链接/卡片/图库/表单/看板（旧页向后兼容）。
4. 数据看板 / 数据视图组件可直接绑定 data.json 集合，实时出数。
5. 点「保存并生成」→ 页面 HTML 经 Worker 写入站点根，前台可访问。
6. 应用详情页可「🌐 生成应用首页」；勾了「进前台导航」的会出现在主站左侧「应用」分组。
7. 已生成的页面可再次「编辑」→ 改内容 →「保存并生成」覆盖更新。
