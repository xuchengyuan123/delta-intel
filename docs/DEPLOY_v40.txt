三角洲情报台 · v40 部署说明（悬浮音乐播放器按音乐台重建）
================================================

问题
----
之前 v38/v39 的悬浮播放器（js/music.js）是另一套实现：用 data.json.music[] 的
QQ outchain iframe 播放，并不是 music.html「音乐台」那套能真正播放的逻辑。用户要求
悬浮播放器严格按代码里 music.html 的音乐台功能来改造。

本次改动（v40）
--------------
1. js/music.js 重写为「音乐台同款」播放器：
   - 多源 Meting API（网易云/QQ音乐/酷狗/酷我/千千），自动选可用源；
   - 真实 <audio> 元素播放（不再是 iframe），与 music.html 完全一致；
   - 后台 data.json.music[] 的歌曲，按「歌名」走音乐台多源搜索拿到可播放链接与封面
     （与音乐台获取链接方式完全一致；无 songid 的曲目也能按歌名搜到，不再跳过）；
   - 默认载入《Unbreakable（坚不可摧）》（与音乐台一致）；
   - 展开悬浮条可搜歌并加入播放列表（多端点回退，结果追加不去覆盖后台歌单）；
   - 上一首 / 播放 / 下一首 / 收起(胶囊)展开；播放结束自动下一首。
2. css/style.css：移除旧的 .music-float / .mf-* 样式，改为与音乐台统一的 .mu-float 体系
   （含 .mu-search/.mu-btn/.mu-list/.mu-card/.mu-cover 等共享类）。
3. music.html：删除与全局重复的内联 .mu-* 播放器样式（改读全局 css/style.css），
   仅保留页面级样式，避免样式漂移。
4. 版本升 v40：全站 ?v=39 -> ?v=40（13 文件），sw.js CACHE=delta-intel-v40，
   js/app.js 注册 sw.js?v=40，data.json changelog 头部加 v40。

注意
----
- js/music.js 只被 index.html（SPA 主站）引入；music.html 是独立页面、自带同款播放器，
  二者不会重复（js/music.js 内有 #muFloat 已存在则跳过的保护）。
- 全站悬浮播放器默认折叠为右下角胶囊，点「展开 ▲」即变完整音乐台（含搜歌与歌单）。
- Meting 公共 API 偶尔不稳定：某源失败时搜索会自动回退其它源；若某首后台歌曲解析失败，
  播放器会提示「无法获取播放链接，换一首试试」，不影响其它歌曲。

部署步骤
--------
1. 上传桌面「三角洲情报台-上传包」整包到 GitHub Pages 仓库根目录。
2. 浏览器打开站点，Ctrl+F5 强刷（清掉 v39 旧缓存）。
3. 右下角应出现音乐胶囊，展开后可搜歌/播放后台歌曲。
