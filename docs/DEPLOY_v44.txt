## 二、Cloudflare Worker（仅当要用工作台 KV 存储时需要）
> 不绑 KV 也能用：未绑定 WORKBENCH 时，工作台会自动回退到 data.json.works，照常工作，只是数据随整包上传覆盖。

### 2.1 创建并绑定 WORKBENCH KV（一次性的事）
```bash
cd cloudflare
wrangler kv namespace create WORKBENCH
# 把返回的 id（一长串十六进制）替换 wrangler.toml 里：
#   [[kv_namespaces]]
#   binding = "WORKBENCH"
#   id = "REPLACE_WITH_YOUR_WORKBENCH_KV_ID"
```

### 2.2 部署 Worker
```bash
cd cloudflare
wrangler deploy
```
部署后 Worker 新增接口（总管理员可用）：
- `GET  /api/admin/works`  → `{works:[...]}`
- `PUT  /api/admin/works`  → `{works:[...]}` body，保存到 WORKBENCH KV

### 2.3 其它可选 KV（历史遗留，按需绑定）
- TEAMS（战队大厅）、ACCESS（访问日志）、SECURITY（安全队列，不绑则复用 ACCESS）等，详见 wrangler.toml 注释与各 DEPLOY_vXX.txt。
